# React component checklist

## UX states
- [ ] loading state
- [ ] empty state
- [ ] error state (actionable message)
- [ ] success state

## Accessibility
- [ ] labels for inputs
- [ ] focus order makes sense
- [ ] keyboard operable
- [ ] headings/landmarks reasonable

## Performance
- [ ] list rendering not pathological
- [ ] expensive computations memoized only if needed
- [ ] handlers not recreated in hot paths unless necessary

## Testing
- [ ] critical behavior covered (unit/E2E)
